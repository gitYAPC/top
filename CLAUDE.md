# AnimeList — трекер просмотренного аниме

Веб-сайт, где пользователь ищет аниме, добавляет в свой список
(смотрю / просмотрено / запланировано / брошено), ставит оценки.
Профили пользователей публичные: /user/{username}.

## Стек

- Python 3.12, FastAPI, uvicorn
- SQLite через SQLAlchemy 2.x (декларативные модели)
- Миграции: Alembic — схема БД меняется ТОЛЬКО через миграции, с первого коммита
- Шаблоны: Jinja2
- Фронт: HTMX (частичные обновления), Alpine.js (мелкий интерактив), Tailwind CSS
- Авторизация: сессии в cookie (starlette SessionMiddleware), пароли — passlib[bcrypt]
- Данные об аниме: Jikan API v4 (https://api.jikan.moe/v4)

## Архитектура

Трёхслойная структура, слои не смешивать:

- `app/routes/` — тонкие роуты: распарсить запрос → вызвать сервис → отдать шаблон.
  Роут не длиннее ~5 строк, без бизнес-логики.
- `app/services/` — вся логика: add_to_list, update_status, get_profile_stats и т.д.
- `app/models.py` — SQLAlchemy-модели.

Весь доступ к Jikan — ТОЛЬКО через модуль `app/anime_source.py`
с интерфейсом: `search(query)`, `get_details(mal_id)`.
Никакой другой код не знает про Jikan. Это позволит потом заменить
источник данных (например, на дамп anime-offline-database), не трогая остальное.

### Jikan: правила работы

- Rate limit: 3 запроса/сек, 60/мин. Между исходящими запросами — задержка,
  на 429 — retry с backoff.
- Кэшировать ВСЁ в таблицу anime: результат поиска и детали тайтла
  сохраняются в БД; повторные обращения идут в свою базу, а не в Jikan.
- `mal_id` — канонический идентификатор аниме во всём проекте (unique в БД).

## Схема БД

```
users:      id, username (unique), password_hash, avatar_url, bio, created_at
anime:      id, mal_id (unique), title, title_english, poster_url,
            episodes, year, synopsis, score_mal, cached_at
ratings: user_id, anime_id, status, score (1-10, nullable), updated_at
            PK (user_id, anime_id)
            status: enum watching / completed / planned / dropped
```

## Роуты (MVP)

- GET  /                       — главная: популярное + поиск
- GET  /search?q=              — результаты поиска (HTMX: живой поиск)
- GET  /anime/{mal_id}         — страница тайтла
- POST /anime/{mal_id}/status  — добавить/сменить статус (HTMX, без перезагрузки)
- POST /anime/{mal_id}/score   — поставить оценку (HTMX)
- GET  /user/{username}        — публичный профиль: список, статистика
- GET  /me                     — свой профиль, редактирование bio/аватара
- GET/POST /register, /login, POST /logout

## Стиль кода

- Минимальный и плоский: без лишних абстракций, фабрик и паттернов «на вырост»
- Без docstrings и декоративных комментариев
- Конфигурация (константы, настройки) — в начале файла или в app/config.py
- Понятные имена вместо комментариев

## Дизайн (референс: Letterboxd × AniList, тёмный минимализм)

Главные герои страницы — постеры аниме. Интерфейс им не мешает.

### Палитра
- Фон страницы: #14181c
- Поверхности (карточки, панели, инпуты): #1c2228
- Границы: #2c3440, тонкие (1px)
- Основной текст: #e3e8ee; вторичный: #9ab
- Акцент (кнопки, ссылки, активный статус): #00c896
- Второй акцент (оценки, звёзды): #f5a623
- Статусы: watching #00c896, completed #4a90d9, planned #9b6dd7, dropped #6b7280

### Типографика и сетка
- Шрифт: Inter (fallback system-ui), заголовки 600, текст 400
- Постеры: строго 2:3, радиус 6px, object-fit: cover
- Сетка карточек: адаптивная, minmax(150px, 1fr), gap 16px
- Много воздуха: контент-контейнер max-w 1200px, щедрые отступы

### Анимации (сдержанные, 150–250ms, ease-out)
- Ховер карточки: подъём translateY(-3px) + рамка акцентного цвета + лёгкая тень
- Появление результатов поиска: fade-in + лёгкий slide-up, stagger 30ms
- Скелетоны (#1c2228, shimmer) при загрузке данных из Jikan
- Смена статуса: кнопка плавно меняет цвет, тост-уведомление (Alpine) снизу справа
- View Transitions API для переходов между страницами (прогрессивно:
  без поддержки — обычная навигация)
- НЕ делать: параллаксы, длинные (>300ms) анимации, анимации ради анимаций

## Деплой (этап 6, не раньше)

- Docker Compose: контейнер приложения + Caddy (автоматический HTTPS)
- Файл SQLite — обязательно в volume (не внутри контейнера!)
- `alembic upgrade head` — в команде запуска контейнера
- Деплой на VPS: git pull && docker compose up -d --build

## План этапов

1. Скелет: структура проекта, модели, Alembic init + первая миграция,
   главная страница с базовым лейаутом и палитрой
2. Авторизация: регистрация, логин, сессии, /me
3. Jikan: anime_source.py, поиск с кэшированием, страница тайтла
4. Списки: user_anime, смена статусов и оценок через HTMX, тосты
5. Профили: /user/{username}, статистика, полировка дизайна и анимаций
6. Деплой: Dockerfile, compose.yml, Caddyfile

Работаем строго по этапам: один этап — проверка — следующий.
